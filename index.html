FEDoc - Farming Game Engine Documentation

Overview

Welcome to FEDoc, the official documentation for the Farming Game Engine built using Unreal Engine 5.6 in C++.

Core Systems

Inventory System

  • InventoryComponent: Manages inventory logic including adding, removing, and item stacking. Fires events on inventory changes to update UI.
  • InventorySlot: Handles logic for item stacking and updating individual slots.
  • InventoryWidget / InventorySlotWidget: Provides visual representation and interactive UI for inventory management. Supports drag-and-drop item movement and quick access.

Player Character

  • FarmerCharacter: Central pawn class for the player, integrating input handling for movement, interactions, harvesting, and combat. Utilizes StatsComponent for managing player attributes such as health and stamina.
  • FarmerController: Custom controller managing player input, camera systems, and interaction logic.

Game Management

  • GM_Farm: Core gameplay manager responsible for maintaining overall game state, including player spawning and initialization.

Items & Resources

  • ItemData: Data-driven approach defining properties for each in-game item, such as icons, stack limits, and interactions.
  • DroppedItem / DroppedTreeLoot: Represents physical items spawned in-world, collectible by player interaction after harvesting or resource depletion.

Environment & Interaction

  • TreeActor: Represents trees in-game, serving as harvestable resources. Implements the DamageableInterface to facilitate harvesting mechanics.
  • DamageableInterface: Standardized interface for destructible or interactive resources, streamlining interaction logic.

Player Stats & Damage

  • StatsComponent: Manages player-specific attributes including health, stamina, damage calculation, and healing. Notifies relevant systems when stats change, enabling dynamic gameplay and UI updates.

Blueprint API

UInventoryComponent

  • AddItem(UItemData* Item, int32 Quantity) - Adds an item to the inventory. (Callable)
  • RemoveItem(UItemData* Item, int32 Quantity) - Removes an item from the inventory. (Callable)
  • HasItem(UItemData* Item, int32 Quantity) - Checks if inventory contains an item. (Callable)
  • ClearInventory() - Clears the inventory. (Callable)
  • GetItemCount(UItemData* Item) - Gets the count of a specific item. (Callable)
  • InventorySlots - The array of inventory slots. (Read Only)

FInventorySlot (Struct)

  • ItemData - The item in this slot. (Read/Write)
  • Quantity - Number of items in this slot. (Read/Write)

UItemData

  • Name - Item display name. (Read Only, Edit Anywhere)
  • Icon - Item icon texture. (Read Only, Edit Anywhere)
  • MaxStackSize - Max stack size. (Read Only, Edit Anywhere)
  • Rarity - Rarity (integer). (Read Only, Edit Anywhere)

UStatsComponent

  • MaxHealth - Maximum health. (Read Only)
  • Health - Current health. (Read Only)
  • MaxStamina - Maximum stamina. (Read Only)
  • Stamina - Current stamina. (Read Only)
  • TakeDamage(float Amount) - Damages the owner. (Callable)
  • Heal(float Amount) - Heals the owner. (Callable)
  • UseStamina(float Amount) - Uses stamina. (Callable)

AFarmerCharacter

  • InventoryComponent - Inventory component (Read Only)
  • StatsComponent - Stats component (Read Only)
  • Interact() - Interact with world. (Callable)
  • Harvest() - Harvest nearby resource. (Callable)
  • UseItem(UItemData* Item) - Use an item. (Callable)
  • DropItem(UItemData* Item, int32 Quantity) - Drop an item. (Callable)

ATreeActor

  • MaxHealth - Tree’s max health (Read Only)
  • Health - Tree’s current health (Read Only)
  • TakeDamage(int32 DamageAmount) - Applies damage. (Callable)

ADroppedItem

  • ItemData - Item contained. (Read Only)
  • Quantity - Number of items. (Read Only)

Workflow

Inventory Management

  • Collect item → Spawn as DroppedItem → Player interaction → Added to InventoryComponent → UI (InventoryWidget) updated automatically.

Resource Harvesting

  • Interact with TreeActor → Apply damage via DamageableInterface → Resource depleted → Spawn DroppedTreeLoot → Collect loot → Inventory updated.

Player Stats Interaction

  • Player actions modify StatsComponent → Health/Stamina updated → UI and gameplay reflect changes dynamically.
