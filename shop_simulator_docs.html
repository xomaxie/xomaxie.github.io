FEDoc - Shop Simulator Game Documentation

Overview

This document describes the architecture and flow for the Shop Simulator system, a general shop game where the player buys items from adventurers, sells to customers, and manages their own inventory and shop layout in Unreal Engine 5.6 (C++).

Game Flow

  1. Game Start / Day Begin
    • Load save or start new game
    • Initialize shop (AShopBuilding), player (AShopPlayerCharacter), and all inventory/components
    • Set current day, time, shop funds, and any pending supply deliveries
  2. Open Shop
    • Player can place/arrange furniture and shelves
    • Stock items from shop inventory onto shelves
    • Order new stock via the computer
  3. Adventurer Arrival & Selling
    • At intervals, spawn AAdventurerNPCs with random loot
    • Adventurers may offer to sell loot to player or browse/shop
  4. Shop Management Loop
    • Player interacts with adventurers (buy/sell, negotiate)
    • Adventurers buy items from shelves if conditions are met
    • Customers proceed to checkout
    • Player manages/restocks shelves, arranges shop, orders supplies
    • Supply deliveries arrive after a delay
  5. End of Day
    • At closing time, all customers leave, shop closes
    • Game saves progress
    • Profit/loss, customer satisfaction, reputation calculated
  6. Progression & Upgrades
    • Spend earnings to unlock new furniture, expand shop, improve reputation
  7. Next Day
    • Advance to next day; repeat from "Open Shop"

Optional/Advanced Events (Anytime)

  • Random events (theft, VIP, inspections, festivals)
  • Special orders or rare supply deliveries
  • Upgrades or shop renovations
